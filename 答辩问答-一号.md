# 答辩问答 - 一号回答

## 问题：这个项目是怎么跟前端连接的？

### 回答要点（建议按顺序讲解）

---

#### 1. 整体模式

我们项目采用 **前后端一体** 的架构：前端页面和后端服务打包在同一个 Spring Boot 应用里，没有单独的前端工程。启动后访问 `http://localhost:8083` 即可使用整个系统。

---

#### 2. 技术栈：Thymeleaf 模板引擎

- 依赖在 `pom.xml` 第 53–55 行：`spring-boot-starter-thymeleaf`
- 前端 HTML 模板放在 `src/main/resources/templates/` 下
- 控制器返回的是模板名称（如 `"login"`、`"main"`），由 Thymeleaf 自动渲染成 HTML 返回给浏览器

---

#### 3. 两种前后端交互方式

**方式一：传统表单 + 页面跳转**

- 登录、成绩录入、用户管理等功能：表单用 `form action="..." method="post"` 提交
- 后端 Controller 用 `@RequestParam` 接收参数，处理完后：
  - 成功：`return "redirect:/main"` 等重定向
  - 失败：`return "login"` 并 `model.addAttribute("error", ...)` 把错误信息带回页面
- 前端用 Thymeleaf 的 `th:text="${error}"` 显示错误
- 典型例子：`LoginController` 第 41–74 行、`login.html` 第 12–24 行

**方式二：AJAX + JSON（AI 助手）**

- AI 聊天需要不刷新页面就更新对话内容，所以用 JavaScript 的 `fetch` 发请求
- 前端 `chat.html` 第 283–303 行：`fetch('/ai/ask', { method: 'POST', ... })`
- 后端 `AIController.askAI`（第 64–94 行）加 `@ResponseBody`，返回 `Map<String, Object>`，Spring 自动转为 JSON
- 前端 `response.json()` 解析后更新页面 DOM，实现实时对话

---

#### 4. 数据如何传到前端

- Controller 通过 `Model` 或 `ModelAndView` 放数据：`model.addAttribute("user", user)`
- 模板里用 `th:text="${user.name}"`、`th:if="${user.role == 'admin'}"`、`th:each="student : ${students}"` 等展示
- 参考：`main.html` 第 14–15 行（用户信息）、第 21–30 行（按角色显示菜单）

---

#### 5. 静态资源

- CSS 放在 `src/main/resources/static/css/style.css`
- 模板里写 `href="/css/style.css"`，Spring Boot 会从 `static/` 目录下发静态文件

---

#### 6. 登录与会话

- 登录成功后，用户信息存入 `HttpSession`
- 各页面的 Controller 通过 `session.getAttribute("user")` 判断是否登录
- 未登录统一 `return "redirect:/login"`，实现鉴权

---

### 一句话总结

> 项目用 Thymeleaf 做服务端渲染，表单提交走传统 MVC 跳转，AI 聊天用 fetch 调 JSON 接口，数据通过 Model 和 Session 传递，静态资源在 `static` 目录下。
