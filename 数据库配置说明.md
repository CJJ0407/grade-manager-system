# 数据库配置说明

当网页上**无法修改用户信息**、或提示“加载失败”时，多半是数据库未正确连接。按下面步骤排查。

## 1. 确认 MySQL 已安装并启动

- **Windows**：服务中查看 “MySQL” 或 “MySQL80” 是否正在运行；或在命令行执行 `mysql -u root -p` 能登录即可。
- **Mac/Linux**：`sudo systemctl status mysql` 或 `brew services list` 查看 MySQL 状态。

## 2. 创建数据库并建表

本系统使用的数据库名为：**grade_management_system**。

1. 用 MySQL 客户端或 Navicat 等工具连接本机 MySQL。
2. 执行项目里的建库建表脚本：
   - 脚本位置：项目根目录下的 **GradeManagemer.sql**（与 GradeManager 文件夹同级）。
   - 在 MySQL 中执行该脚本，会创建数据库 `grade_management_system` 以及 `user`、`subject`、`exam`、`score` 等表及初始数据。

若找不到该文件，可在 MySQL 中手动执行：

```sql
CREATE DATABASE IF NOT EXISTS grade_management_system CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE grade_management_system;
-- 然后执行 GradeManagemer.sql 中从 CREATE TABLE 开始的所有语句
```

## 3. 核对 application.properties 配置

打开 `src/main/resources/application.properties`，确认与你的 MySQL 一致：

| 配置项 | 说明 | 默认示例 |
|--------|------|----------|
| `spring.datasource.url` | 地址、端口、数据库名 | `jdbc:mysql://localhost:3306/grade_management_system?serverTimezone=UTC&useUnicode=true&characterEncoding=utf-8&allowPublicKeyRetrieval=true` |
| `spring.datasource.username` | MySQL 用户名 | `root` |
| `spring.datasource.password` | MySQL 密码 | `123456` |

- 若 MySQL 端口不是 3306，把 url 里的 `3306` 改成实际端口。
- 若密码不是 `123456`，请改成你自己的 MySQL 密码。

## 4. 重启应用并查看日志

1. 保存 `application.properties` 后，**重启** Spring Boot 应用。
2. 启动时若报错：
   - **Unknown database 'grade_management_system'**：说明数据库未创建，请完成第 2 步。
   - **Access denied for user 'root'@'localhost'**：说明用户名或密码错误，请修改 `username`/`password`。
   - **Communications link failure**：说明连不上 MySQL，检查 MySQL 是否启动、端口是否正确、防火墙是否拦截。

修改用户信息时若仍失败，请查看控制台或日志文件中的 **“更新用户失败”** 或 **“更新个人信息失败”** 后面的异常信息，根据报错内容对照上面几条排查。
